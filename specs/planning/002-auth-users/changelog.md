- Architectural fix during testing: `JwtAuthenticationMiddleware` originally let `AuthenticationRequired` propagate, expecting `AppErrorMiddleware.process_exception` to redirect — but Django only calls `process_exception` for view-raised errors, not middleware-raised ones, so an invalid cookie surfaced as a 500. - Code review (code-reviewer agent, fresh context) — assessment "Needs fixes before proceeding". Important findings, all addressed:
  - **`OrmUserRepository.upsert` was clobbering cached SIGA fields** on JWT-only re-logins (regression of acceptance criterion: "falls back to whatever was previously stored"). The in-memory fake hid the bug because it preserved values, while the ORM impl wrote `""`. Fixed: ORM now mirrors the `semestre` rule for `full_name` and `programa` — empty string / `None` is interpreted as "no information" and never overwrites cached values. Regression guard added in `test_upsert_preserves_cached_siga_fields_on_jwt_only_relogin`.
  - **`hydrate_from_siga` was letting SIGA overwrite the user's email**. Auth provider is the security-of-record for identity; SIGA is enrichment. Fixed: `hydrate_from_siga` now passes `email=existing.email`, never `profile.email`. Test updated to assert the JWT email survives a SIGA hydration with a different email.
  - **`CallbackView` was calling `decode_jwt` twice on empty token** (once to force the raise, once "for real"). Replaced with an explicit `raise AuthenticationRequired("Token ausente...")` and a single decode call.
  - **`AppErrorMiddleware` redirected to `LOGIN_REDIRECT_URL` while `JwtAuthenticationMiddleware` redirected to `LOGIN_URL`**. Both now use `settings.LOGIN_URL` (Django's documented setting for "where to send anonymous users"). Touches initiative 001's `_shared/middleware/error_handler.py` — minor, no test changes needed.
