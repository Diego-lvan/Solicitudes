## 2026-04-25 — Review round 1 (code-reviewer agent → /receive-review)

Reviewer assessment was "needs changes before proceeding". Acted on every Important item; deleted unused glue spotted in Suggestions.

- **Tipo `<select>` selected-option bug**: comparison was `filter.tipo_id (UUID) == option.value (str)` → always False, so the active tipo silently reset to "Todos" on every reload. Fixed by exposing `selected_estado` / `selected_tipo_id` / `selected_role` (pre-stringified active values) from `views/_helpers.py:filter_form_choices(filter)`; the template now compares strings on both sides. Regression test added: `reportes/tests/test_views.py::test_dashboard_tipo_filter_marks_option_selected`.
- **Spanish labels**: `_helpers.py:filter_form_choices` now returns `(value, Estado.display_name)` and an explicit `_ROLE_DISPLAY` map (`Control Escolar` / `Responsable de Programa` / `Docente`). Dashboard / list / PDF templates render `{% include "solicitudes/_partials/_estado_badge.html" with estado=... %}` (or `estado.display_name` in the print PDF) instead of raw enum identifiers.
- **Exporter pagination round trips**: removed the per-chunk `count()` cost. New `SolicitudRepository.iter_for_admin(filters, chunk_size=500)` (Django `.iterator(chunk_size=...)`) + `LifecycleService.iter_for_admin` + `ReportService.iter_for_admin`. CSV exporter now writes rows directly from the iterator; PDF exporter walks the iterator until it either exhausts or hits `_PDF_ROW_CAP + 1` rows. The `PageRequest.page_size` cap stays at 100; exports never go through it.
- **PDF truncation single source of truth**: removed the `len(rows) < cap` belt-and-suspenders. The exporter now sets `truncated=True` if and only if the iterator emitted more than `_PDF_ROW_CAP` rows; if truncated, `rows=[]` is passed to the template and the Spanish "Demasiados registros — use la exportación CSV" notice fully replaces the row table. Test added: `test_pdf_truncates_when_more_than_cap_rows_match` (uses `monkeypatch` on `render_pdf` and a stub `ReportService` to avoid materializing real rows).
- **Single `DefaultReportService` per export request**: `dependencies.get_csv_exporter` / `get_pdf_exporter` now accept an optional `report_service` arg; the export views construct one `get_report_service()` and pass it in so the dashboard call and the iter call share one composed graph.
- **`docker-compose.dev.yml` reverted** to its original (host port 80/443 for nginx, 5432 for db). `docker-compose.override.yml` now uses Compose v2.24+ `!override` to replace the ports lists (nginx → 8081/8444, db → none); verified the worktree stack still binds on 8081/8444 and the main checkout's port mappings are intact.
- **Suggestions taken**: deleted unused `reportes/exceptions.py` (`InvalidReportFilter` was never raised); deleted unused `_helpers.py:get_report_service_dep`; switched `templates/reportes/list.html` pagination to use `page.has_prev` / `page.has_next` (Pydantic computed fields).
- **Multi-line Django comment leak**: caught visually in the post-edit screenshot. `{# ... #}` is single-line only; replaced with `{% comment %}…{% endcomment %}` in `_filter_form.html`.

Verified after fixes: `ruff check .` clean, `pytest` 387 passed (added 2 tests), Tier 2 reportes E2E passes; desktop and mobile screenshots re-captured at `/tmp/screenshots-009/` and visually reviewed (Spanish badges render, no template-comment leak, mobile reflow intact).
