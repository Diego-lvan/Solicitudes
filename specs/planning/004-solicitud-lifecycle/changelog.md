- **Code review round 1** (code-reviewer agent, fresh-context). Assessment: "Fix Important issues before proceeding." 4 Important + several Suggestions raised. Triage and response:
  - **Important — silent observaciones loss in revision views (`take.py`, `finalize.py`, `cancel.py`)**: agreed; views read `cleaned_data["observaciones"] if form.is_valid() else ""`, silently discarding invalid input. Fixed: on `not form.is_valid()` flash `messages.error("Las observaciones no son válidas.")` and redirect back to the detail without invoking the transition.
  - **Important — `assert` in `_shared/request_actor.py`**: agreed; `python -O` strips asserts, weakening the security-adjacent guard. Replaced with `raise AuthenticationRequired(...)` so the global error middleware redirects to the login flow exactly as it does for the mixin.
  - **Important — silent file discard in intake**: agreed with the proposed mitigation; until 005 lands, the user now sees `messages.warning("Los archivos adjuntos aún no se almacenan; tu solicitud se registró sin ellos. ...")` in addition to the existing `WARNING` log line.
  - **Important — `_authorize` raises `InvalidStateTransition` on unknown action**: agreed; semantics were wrong (that exception is for state mismatches, not unknown actions). Replaced with `raise ValueError(f"Unknown action: {action!r}")` since the branch is unreachable from views and indicates a programming bug.
  - **Suggestion — `tipo_id` filter unreachable from `mis_solicitudes` UI**: removed from `_read_filters` rather than adding the `<select>`; lightweight alumno volume doesn't yet justify the extra control. Documented in the view docstring.
  - **Suggestion — `Estado` display name uses `|title` which renders `EN_PROCESO` as `En_Proceso`**: added a `display_name` property on `Estado` (mapped via `_ESTADO_DISPLAY` to the proper Spanish label) and switched all template renderings to `{{ estado.display_name }}` (`mis_solicitudes.html`, `revision/queue.html`, `_estado_badge.html`).
  - **Suggestion — `FolioCollision` and `ComprobanteRequired` are dead code**: kept as future-proofing surfaces; added docstrings explaining the conditions under which they would be raised (sequence-collision allocator and service-level comprobante validation respectively).
  - **Suggestion — wrap `notify_*` and `audit.write` in try/except**: pushed back. With the current `NoOpNotificationService` no exception path exists, and when 007 lands the real adapter it should decide its own retry/swallow policy at the adapter boundary rather than 004 hard-coding a `try/except` here. Will revisit when 007 ships.
  - **Suggestion — `repo._rows.clear()` private access in property test**: kept; in-memory test fakes are intentionally low-ceremony and the access pattern is confined to the test module.
  - **Suggestion — dedicated `intake_service` unit tests**: deferred. The reviewer marked it non-blocking; create flow is covered through view + Tier-1 e2e tests, and the snapshot integrity invariant is exercised in `test_e2e_tier1.py::test_snapshot_integrity_when_tipo_label_changes_after_creation`. Will add direct service-level coverage if/when the create flow gains additional branches.
  - **Suggestion — port-file naming (`*_port.py`)**: the convention isn't in the architect rules doc. Decision: keep the current naming for this initiative and codify it in `.claude/rules/django-code-architect.md` during initiative closeout (after `/review` runs cleanly). Alternative would be moving them into `services/<name>/{interface,implementation}.py`, but they aren't services this app *implements*; they are outbound contracts whose adapters live in another feature.
