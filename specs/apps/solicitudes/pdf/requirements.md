# pdf — Requirements

> WHAT this feature does and WHY. No implementation details. Shipped by initiative **006 — PDF Generation**.

## Purpose

Turn a **plantilla** (admin-managed HTML+CSS document template with `{{ var }}` placeholders) plus a **solicitud** into a downloadable PDF. The plantilla is the institution's official document layout (constancia, oficio, comprobante, etc.); the PDF is what the solicitante or personal hands off to other offices.

This feature satisfies **RF-03** (plantillas with variables, optional per-tipo) and provides the admin surface to create, edit, preview, and deactivate plantillas.

## Why server-side rendering on demand

Generated bytes are not stored. Every download re-renders from the live `Solicitud` + `PlantillaSolicitud` so the output is always consistent with the current source data — there's no chance of a stale PDF lying around with outdated values, and no storage cost for an artifact that's cheap to regenerate.

The trade-off is render time per request. WeasyPrint is fast enough for the project's load profile, and the architecture keeps re-render an option (a future cache layer can be added without changing the service contract).

## In scope

- The **`PlantillaSolicitud` model**: an admin-editable Django-template HTML body, optional CSS, name/description, soft-delete flag, timestamps.
- An **admin CRUD surface** for plantillas: list, create, detail with inline preview, edit, deactivate.
- A **synthetic-context preview** so admins can iterate on a plantilla without needing a real solicitud — same context keys, placeholder values.
- A **per-solicitud PDF download** at `/solicitudes/<folio>/pdf/`, with a structured authorization matrix (admin / personal / owner-finalized).
- A **variable resolution layer** that maps a `SolicitudDetail` + the resolved `UserDTO` of the solicitante onto a context dict the plantilla renders against. Context keys cover: `solicitante` (matrícula, nombre, email, programa, semestre), `solicitud` (folio, estado, tipo nombre, timestamps), `valores` (slug-keyed snapshot field values), `now`, and `firma_lugar_fecha` (Spanish-localized).
- A **deterministic render contract**: identical inputs under a frozen clock produce byte-identical PDFs (within an environment).
- A **plantilla selector** on the tipo form so admins can attach a plantilla to a tipo; tipos without a plantilla render no PDF affordance.

## Out of scope

- Storing rendered bytes. Every download re-renders.
- Per-PDF watermarks, digital signatures, certificate sealing, QR validation. Possible future work; not v1.
- A WYSIWYG editor for plantillas. Admins write Django-template HTML+CSS directly. The detail page's iframe preview is the authoring feedback loop.
- Mass / bulk PDF export. Reports' CSV is the bulk path; per-solicitud PDF is on-demand only.
- Asset bundling. Plantillas may reference `/static/...` paths during a single deployment, but cross-deployment byte stability requires `data:` URIs (documented in design).
- Plantilla versioning / history. Editing a plantilla edits-in-place; previously-generated PDFs are not re-rendered or stored, so there's nothing to version.

## Non-functional requirements

- **Determinism within an environment.** Two renders of the same solicitud + plantilla under a frozen clock produce byte-identical PDFs. The WeasyPrint `pdf_identifier` hook + a fixed clock pin `/ID`, `/CreationDate`, and `/ModDate`. Cross-deployment byte stability requires plantillas to embed assets as `data:` URIs (documented in design).
- **No persistence of rendered bytes.** Every download is a fresh render from current source data.
- **Bounded surface.** WeasyPrint is reached only through `_shared/pdf.render_pdf`. No other call site in the codebase imports WeasyPrint.
- **Cross-feature reads, never writes.** This feature consumes `LifecycleService` (read) and `UserService` (read); it never advances state, never invokes notifications, never edits the solicitud or its archivos.
- **Spanish UI copy** end-to-end (admin pages, error messages, plantilla preview helper text). Code identifiers in English.
- **Accessibility.** Admin forms expose required indicators, `role="alert"` errors, monospace textareas for HTML/CSS, and a primary-right-of-cancel button order. The plantilla preview iframe sets `X-Frame-Options: SAMEORIGIN`.
