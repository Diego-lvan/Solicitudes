# Global Explorations

In-flight `/brainstorm` drafts that don't yet have a clear feature home.

When `/brainstorm` is invoked for an idea whose target app/feature isn't decided yet, it parks the draft `requirements.md` here as `YYYY-MM-DD-<topic>.md`. `/plan` later moves the content into the proper `apps/<app>/<feature>/requirements.md` location once the module decision is made.

**Naming:** `YYYY-MM-DD-<short-topic-slug>.md`

**Lifecycle:** drafts here are temporary. They are either promoted to `apps/<app>/<feature>/requirements.md` by `/plan`, or discarded if the idea is dropped. Don't accumulate stale drafts — clean up when an idea is settled or abandoned.
