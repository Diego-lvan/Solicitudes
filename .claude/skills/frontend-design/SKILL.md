# Frontend Design — Django Templates + Bootstrap 5

## Core principle

**Restraint over flourish.** Institutional software (universities, government, internal tools) earns trust through clarity, density, and consistency — not through gradients, glow, or animation. Default to looking like a serious tool, not a marketing landing page.

If a choice would look at home on a SaaS landing page generated by an AI prompt, reject it.

## The "AI look" — what to avoid

These are the dead giveaways. Refuse them by default:

| Tell | Why it reads as AI | Use instead |
|------|---------------------|-------------|
| Purple→pink or blue→purple gradients (especially on hero, buttons, headings) | Default output of every generator | One flat institutional brand color (UAZ green for this project) |
| Glassmorphism / `backdrop-filter: blur` cards | Overused 2023 trope | Solid `bg-white` with a 1px border or subtle shadow |
| Emoji as section icons (🚀 ⚡ ✨ 📊) | Hallmark of generated content | Real icon set (Bootstrap Icons or Lucide), monochrome |
| Generic structure: hero → 3 feature cards → testimonials → CTA | The Lovable/v0 default | Page structure driven by the actual task (form, table, detail view) |
| Centered hero with huge headline + tiny subtitle + 2 buttons | Marketing template muscle memory | Real apps start with content (a list, a form), not a pitch |
| "Get started" / "Learn more" / "Powered by AI" copy | Generated filler | Action-specific Spanish: "Crear solicitud", "Ver mis solicitudes" |
| Floating animated blobs / mesh gradients in background | Pure decoration | Plain background |
| Rounded-2xl everywhere + heavy drop shadows | "Modern AI" cliché | Bootstrap default `rounded` (4px) or `rounded-1`; shadows only on overlays/modals |
| Lucide/Heroicons stroke icons next to every label | Decoration without function | Icons only where they aid scanning (status, actions in tables) |
| Three-color "feature card" sets where each card has its own pastel tint | Auto-generated palette padding | One neutral card style. Color carries meaning (status), not decoration |

**Rule:** Color and motion must encode information. If they're decorative, remove them.

## Visual language for this project

This is a UAZ academic system. The visual identity should read as: **government/university administrative tool**, not startup.

- **Brand color:** UAZ institutional green as the single accent. Used for primary actions, active nav, focus rings. Everywhere else: neutrals.
- **Neutrals do the work:** white background, `#212529` text (Bootstrap default), `#6c757d` muted, `#dee2e6` borders. ~85% of the UI is neutral.
- **Status colors** (Bootstrap semantic) carry meaning only:
  - `success` (green) — approved / completed
  - `warning` (yellow) — pending / requires attention
  - `danger` (red) — rejected / errors
  - `info` (blue) — informational
  - `secondary` (gray) — drafts / cancelled
- **Density > whitespace.** Government users scan tables. Don't pad rows like a Stripe dashboard.

## Typography

- **Single font family.** System stack (`-apple-system, Segoe UI, Roboto, ...`) is fine. Or Inter. Never two display fonts.
- **Scale (Bootstrap defaults are good):** `1rem` body, `h1: 2.5rem`, `h2: 2rem`, `h3: 1.75rem`, `h4: 1.5rem`. Don't invent custom sizes.
- **Weights:** 400 body, 500 emphasis, 600/700 headings. Avoid 300 (thin) — fails contrast and looks dated.
- **Line height:** 1.5 for body, 1.2–1.3 for headings.
- **Spanish text:** all user-facing copy in Spanish, including form errors, empty states, button labels.

## Spacing rhythm

Use Bootstrap's spacing scale (`p-0` through `p-5`, `gap-*`, `mb-*`). Don't write custom margins.

- Section vertical rhythm: `mb-4` between blocks, `mb-5` between major sections.
- Form fields: `mb-3` between fields.
- Card body: `p-3` or `p-4`. Never `p-5` for content cards (that's hero territory).
- Tables: default Bootstrap padding. Don't add custom `cellpadding`.

## Layout patterns (Django template structure)

Inheritance per the project's `django.md` rule:

```
base.html                    → <html>, <head>, navbar, footer, blocks
  {app}/base_{app}.html      → optional app-level layout (sidebar, breadcrumbs)
    {app}/{view}.html        → page-specific content
```

Reusable pieces go in `templates/components/` and are included with `{% include "components/x.html" with arg=value %}`.

**Standard page skeleton:**

```django
{% extends "solicitudes/base_solicitudes.html" %}
{% block title %}Mis solicitudes{% endblock %}

{% block content %}
<div class="container py-4">
  {% include "components/breadcrumbs.html" with items=breadcrumbs %}

  <div class="d-flex justify-content-between align-items-center mb-4">
    <h1 class="h3 mb-0">Mis solicitudes</h1>
    <a href="{% url 'solicitudes:create' %}" class="btn btn-primary">
      <i class="bi bi-plus-lg me-1"></i> Nueva solicitud
    </a>
  </div>

  {# content #}
</div>
{% endblock %}
```

Notes:
- `container` (not `container-fluid`) for content pages — keeps line length readable.
- `container-fluid` only for dashboards / data tables that need full width.
- Page title uses `h1` semantically but `.h3` visually — h1 sized like a marketing hero is wrong for an internal tool.
- Action button placed in the page header, not floating.

## Responsive standards

Bootstrap 5 is mobile-first. Design for `xs` (≤576px) first, then enhance.

**Breakpoints (memorize):** `sm 576` · `md 768` · `lg 992` · `xl 1200` · `xxl 1400`.

**Required behaviors:**

1. **Navbar collapses** to hamburger below `lg`. Use `navbar-expand-lg`.
2. **Tables on mobile:** wrap in `.table-responsive`. For dense admin tables, hide low-priority columns at `<md` with `d-none d-md-table-cell`. Don't try to cram 8 columns into 375px.
3. **Forms:** single column on mobile, two columns from `md` up via `col-md-6`. Labels on top (not left-aligned floating) — works at every width.
4. **Buttons:** full-width (`w-100`) on mobile for primary actions in forms. Inline on `md+`.
5. **Touch targets:** minimum 44×44px (WCAG 2.5.5 AAA, but honor it). Bootstrap's default button height meets this; don't override smaller.
6. **No horizontal scroll** at 320px width except inside `.table-responsive`.
7. **Test at 200% zoom** — content must reflow without loss of function (WCAG 1.4.10).

## Accessibility — WCAG 2.2 AA (non-negotiable)

For an institutional/government-adjacent system, AA is the floor.

- **Contrast 4.5:1** for body text, **3:1** for large text (≥18pt or 14pt bold) and UI components/borders. Never put white text on `bg-warning` (yellow) — fails contrast. Use `text-dark`.
- **Semantic HTML:** `<button>` for actions, `<a>` for navigation, `<table>` for tabular data, `<th scope="col">`, `<label for>` on every input. No `<div onclick>`.
- **Focus visible:** never `outline: none` without a replacement. Bootstrap's focus ring is fine; if customizing, keep a 2px ring with 3:1 contrast against the background.
- **Form labels:** every input has a `<label>` (not just placeholder). Errors associated via `aria-describedby`. Required fields marked in label, not by color alone.
- **Skip link:** `<a href="#main" class="visually-hidden-focusable">Saltar al contenido</a>` as the first focusable element.
- **Headings in order:** one `h1` per page, no skipping levels.
- **Alt text:** every meaningful `<img>`. Decorative images: `alt=""`.
- **Status messages:** Django messages container should have `role="status"` or `aria-live="polite"`.
- **No CAPTCHAs without alternative** (WCAG 2.2 SC 3.3.8 — Accessible Authentication).
- **Consistent help** (WCAG 2.2 SC 3.2.6): support/contact link in the same place on every page.
- **Color is never the only signal:** status badges include text ("Aprobada"), not just a green dot. Form errors include an icon or text, not just red border.

## Component patterns

### Forms

```django
<form method="post" novalidate>
  {% csrf_token %}
  <div class="mb-3">
    <label for="id_titulo" class="form-label">
      Título <span class="text-danger" aria-hidden="true">*</span>
    </label>
    {{ form.titulo|add_class:"form-control" }}
    {% if form.titulo.errors %}
      <div class="invalid-feedback d-block" role="alert">
        {{ form.titulo.errors.0 }}
      </div>
    {% endif %}
    {% if form.titulo.help_text %}
      <div class="form-text">{{ form.titulo.help_text }}</div>
    {% endif %}
  </div>

  <div class="d-flex gap-2 justify-content-end">
    <a href="{% url 'solicitudes:list' %}" class="btn btn-outline-secondary">Cancelar</a>
    <button type="submit" class="btn btn-primary">Guardar</button>
  </div>
</form>
```

- Primary action on the right, secondary/cancel to its left. Mobile: stack with primary on top.
- Never use `<input type="reset">`.
- `novalidate` on the form — let Django/server validate; HTML5 popups are inconsistent.

### Tables (the workhorse of admin UIs)

```django
<div class="table-responsive">
  <table class="table table-hover align-middle">
    <thead>
      <tr>
        <th scope="col">Folio</th>
        <th scope="col">Tipo</th>
        <th scope="col" class="d-none d-md-table-cell">Fecha</th>
        <th scope="col">Estado</th>
        <th scope="col" class="text-end">Acciones</th>
      </tr>
    </thead>
    <tbody>
      {% for s in solicitudes %}
      <tr>
        <td><a href="{% url 'solicitudes:detail' s.folio %}">{{ s.folio }}</a></td>
        <td>{{ s.tipo.nombre }}</td>
        <td class="d-none d-md-table-cell">{{ s.created_at|date:"d/m/Y" }}</td>
        <td>{% include "components/badge_estado.html" with estado=s.estado %}</td>
        <td class="text-end">…</td>
      </tr>
      {% endfor %}
    </tbody>
  </table>
</div>
```

- Use `.table-hover` not `.table-striped` for dense data — striping fights with status colors.
- Numeric / action columns: `text-end`.
- Always provide an empty state (`{% empty %}`) — never show a blank table.

### Status badges

One component, one source of truth:

```django
{# components/badge_estado.html #}
{% if estado == 'BORRADOR' %}<span class="badge text-bg-secondary">Borrador</span>
{% elif estado == 'PENDIENTE' %}<span class="badge text-bg-warning">Pendiente</span>
{% elif estado == 'APROBADA' %}<span class="badge text-bg-success">Aprobada</span>
{% elif estado == 'RECHAZADA' %}<span class="badge text-bg-danger">Rechazada</span>
{% endif %}
```

Use `text-bg-*` (Bootstrap 5.2+) — handles contrast automatically.

### Empty states

Don't leave blank screens. But don't make them feel like marketing either:

```django
<div class="text-center py-5">
  <i class="bi bi-inbox fs-1 text-muted" aria-hidden="true"></i>
  <p class="mt-3 mb-2">No tienes solicitudes registradas.</p>
  <a href="{% url 'solicitudes:create' %}" class="btn btn-primary btn-sm">Crear la primera</a>
</div>
```

One icon, one sentence, one action. No illustrations of people pointing at boxes.

### Cards

Use `<div class="card">` with `.card-body`. Avoid `.card-header` + `.card-footer` unless the structure needs all three. Don't apply `shadow-lg` — `shadow-sm` at most, and only when the card needs to lift off a tinted background.

## Customization (Bootstrap 5 the right way)

Override CSS variables in a single `static/css/theme.css` loaded **after** Bootstrap:

```css
:root {
  --bs-primary: #006837;       /* UAZ green */
  --bs-primary-rgb: 0, 104, 55;
  --bs-link-color: #006837;
  --bs-border-radius: 0.25rem; /* keep modest, not 1rem */
}

/* Make focus visible without being garish */
:focus-visible {
  outline: 2px solid var(--bs-primary);
  outline-offset: 2px;
}
```

Don't fork Bootstrap's SCSS unless you need to change something CSS variables can't reach. For most projects, variables + a few utility overrides are enough.

## Quick checklist before merging UI work

- [ ] No purple/pink gradients, no glassmorphism, no emoji as icons
- [ ] All user-facing text in Spanish
- [ ] Works at 320px width with no horizontal scroll
- [ ] Navbar collapses below `lg`
- [ ] Tables wrapped in `.table-responsive`; low-priority columns hidden on mobile
- [ ] Every form input has a `<label>`; errors readable by screen reader
- [ ] Focus ring visible on every interactive element
- [ ] Body text ≥4.5:1 contrast; status badges use text + color, never color alone
- [ ] One `h1` per page, headings in order
- [ ] Empty states for every list/table
- [ ] Skip-to-content link present on `base.html`
- [ ] Tested at 200% zoom

## Common mistakes

| Mistake | Fix |
|---------|-----|
| Using `placeholder` as the only label | Add `<label>`; placeholder is an example, not a label |
| `btn btn-primary` on every button on the page | One primary per view (the main action). Others: `btn-outline-secondary` or `btn-link` |
| Centering everything | Forms ≤600px wide centered is fine. Tables, dashboards: left-aligned, full container width |
| `container-fluid` everywhere | Default to `container`. Reading line length matters |
| Inventing a new gray | Use Bootstrap's `text-muted`, `border`, `bg-light`. Don't pick `#f3f4f6` from a Tailwind tutorial |
| Animating page transitions | Don't. Government users hate it and `prefers-reduced-motion` users will see it broken anyway |
| Status conveyed only by color | Add the word ("Aprobada"), an icon, or both |
