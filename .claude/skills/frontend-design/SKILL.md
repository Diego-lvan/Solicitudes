# Frontend Design — Django Templates + Tailwind v4 + Alpine.js + Lucide

## Core principle

**Restraint over flourish.** Institutional software (universities, government, internal tools) earns trust through clarity, density, and consistency — not through gradients, glow, or animation. Default to looking like a serious tool, not a marketing landing page.

If a choice would look at home on a SaaS landing page generated by an AI prompt, reject it. The aesthetic target is **shadcn / Vercel / Linear** — pure monochrome, hairline borders, Inter, mid radius, generous-but-dense spacing.

## The "AI look" — what to avoid

| Tell | Why it reads as AI | Use instead |
|------|---------------------|-------------|
| Purple→pink or blue→purple gradients | Default output of every generator | Pure monochrome (`zinc-950` text, `zinc-900` primary) |
| Glassmorphism / `backdrop-filter: blur` cards | Overused 2023 trope | `bg-white` with a `border border-zinc-200` (1px hairline) |
| Emoji as section icons (🚀 ⚡ ✨) | Hallmark of generated content | Lucide icons via `{% lucide "name" %}` |
| Generic structure: hero → 3 feature cards → testimonials → CTA | The Lovable/v0 default | Page structure driven by the actual task (form, table, detail) |
| Centered hero with huge headline + 2 buttons | Marketing template muscle memory | Real apps start with content (a list, a form), not a pitch |
| Floating animated blobs / mesh gradients | Pure decoration | Plain background |
| `rounded-2xl` everywhere + heavy drop shadows | "Modern AI" cliché | `rounded-md` (6px) default, `rounded-lg` (8px) for cards/modals; `shadow-sm` only on overlays |
| Three-color "feature card" sets where each has its own pastel tint | Auto-generated palette padding | One neutral card style; color encodes status (success/warning/danger), not decoration |

**Rule:** Color and motion must encode information. If they're decorative, remove them.

## Design tokens (single source of truth)

The canonical token block lives in `app/static/css/app.css` under `@theme`. Adjust there, never hard-code colors in templates.

```css
@theme {

  --color-bg-subtle:     #fafafa;
  --color-bg-muted:      #f4f4f5;       /* zinc-100 */
  --color-border:        #e4e4e7;       /* zinc-200 — hairlines */
  --color-border-strong: #d4d4d8;       /* zinc-300 — inputs */
  --color-text:          #09090b;       /* zinc-950 — body */
  --color-text-muted:    #71717a;       /* zinc-500 */
  --color-text-subtle:   #a1a1aa;       /* zinc-400 — placeholders */
  --color-primary:       #18181b;       /* zinc-900 — primary buttons */
  --color-primary-hover: #000000;
```

## Build pipeline

Tailwind standalone CLI runs **inside the dev `web` container** — no Node on host. The dev compose `web` service runs `tailwindcss --watch=always` in the background alongside `runserver`. Two Make targets:

```bash
make css         # one-shot minified build (used in CI before collectstatic)
make css-watch   # foreground watcher (`--watch=always` survives backgrounding)
```

`app/static/css/app.css` is the input. `app/static/css/app.build.css` is the generated output (gitignored). The pipeline scans `templates/**/*.html` and `**/*.py` for class names via `@source` directives.

## Typography

- **Inter Variable**, self-hosted from `app/static/fonts/Inter/InterVariable.woff2`. Preloaded in `base.html`.
- Stylistic sets enabled: `font-feature-settings: "cv02","cv03","cv04","cv11"`.
- Slight negative tracking on body (`-0.005em`) and headings (`-0.02em`).
- Headings `font-weight: 600`; body 400; emphasis 500. **Never use 300** (thin) — fails contrast and looks dated.

## Spacing rhythm

Use the Tailwind spacing scale. Defaults:

- Section vertical rhythm: `mb-4` between blocks, `mb-6`/`mb-8` between major sections.
- Form fields: `space-y-1.5` inside one field, `space-y-5` between fields.
- Card padding: `p-5` standard. Never `p-8` for content cards.
- Tables: `px-4 py-3` per cell.

## Layout

Standard authenticated page skeleton (provided by `base.html`):

```
<navbar>            ← components/navbar.html (sticky top-0)
<sidebar> | <main>  ← components/sidebar.html on lg+, drawer on mobile
<footer>            ← inline in base.html
```

Page templates extend `base.html` and put content in `{% block content %}`. The default `<main>` wrapper centers in `max-w-7xl mx-auto px-4 md:px-6 py-6`.

## Component partials catalogue

All in `app/templates/components/`. Each is a self-contained `{% include %}` partial taking parameters via `with`.

| Partial | Purpose | Notable args |
|---|---|---|
| `lucide_sprite.html` | Inlines the SVG sprite (one per page, in `base.html`) | — |
| `button.html` | All button variants × sizes; supports `<a>` mode via `href` | `variant`, `size`, `label`, `icon`, `href` |
| `card.html` | Card surface with optional title + body + footer | `title`, `body_html`, `footer_html` |
| `input.html` | Text input + label + hint + error association | `field` OR `name/id/label/value/type` |
| `select.html`, `textarea.html`, `checkbox.html`, `radio.html` | Form controls | same dual-mode pattern as `input.html` |
| `badge_estado.html` | Status pill keyed off Estado enum (text + color, never color-only) | `estado` |
| `breadcrumbs.html` | `aria-current="page"` on last item; lucide chevron separators | `items` |
| `sidebar.html` | Persistent left rail with role-aware nav | `variant="mobile"` for offcanvas mount |
| `offcanvas_drawer.html` | Mobile sidebar drawer (Alpine `x-show`, escape, backdrop) | `open_state_var` |
| `modal.html` | Native `<dialog>` + Alpine state (preferred) | `id`, `title`, `body_html`, `footer_html`, `open_state_var`, `size` |
| `dropdown.html` | Alpine `@click.outside` + escape | `trigger_html`, `body_html`, `align`, `width` |
| `toast.html` | `aria-live="polite"` + auto-dismiss | `variant`, `message`, `timeout` |
| `alert.html` | Static alert / banner (`role="alert"` for `danger`) | `variant`, `title`, `message`, `icon`, `dismissible` |
| `pagination.html` | Prev/Next + page indicator | `page` (Pydantic `Page` DTO), `param_name` |
| `empty_state.html` | Icon + sentence + CTA | `icon`, `title`, `description`, `action_url`, `action_label` |
| `chip_input.html` | Chip-pills + free-text input backed by hidden CSV | `name`, `id`, `value`, `placeholder`, `aria_label` |
| `navbar.html` | Top bar with brand mark + user menu + drawer trigger | — |

## Iconography

Lucide via the `{% lucide %}` template tag (`app/_shared/templatetags/lucide.py`):

```django
{% load lucide %}
{% lucide "plus" %}
{% lucide "trash-2" class="size-5 text-red-600" %}
{% lucide "check" label="Listo" %}        {# accessible label, sets role=img #}
```

Decorative icons render with `aria-hidden="true"`. Icons that carry meaning take `label="..."` (Spanish), which sets `role="img"` and `aria-label`.

The vendored sprite lives at `app/static/vendor/lucide/sprite.svg` (one symbol per icon). Add a new icon by dropping its SVG into `vendor/lucide/icons/` and re-running the sprite-build script (see `MAPPING.md` in that directory). Keep the vendored set lean — only add icons that are actually used.

## Alpine.js patterns

Alpine v3 (~15 KB) loaded `defer` in `base.html`, plus `@alpinejs/collapse` and `@alpinejs/focus` plugins. **Use Alpine for declarative interactivity; avoid bespoke JS.**

Common patterns:

```html
{# Dropdown #}
<div x-data="{ open: false }" @click.outside="open = false" @keydown.escape.window="open = false">
  <button @click="open = !open" :aria-expanded="open">Menu</button>
  <div x-cloak x-show="open" x-transition>...</div>
</div>

{# Modal — prefer native <dialog> #}
<dialog x-data x-effect="modalOpen ? $el.showModal() : $el.close()"
        @close="modalOpen = false" @keydown.escape="modalOpen = false">...</dialog>

{# Mobile drawer #}
<div x-cloak x-show="drawerOpen" x-transition.opacity>
  <div class="fixed inset-0 bg-zinc-950/40" @click="drawerOpen = false"></div>
  <aside x-transition:enter="transform" x-transition:enter-start="-translate-x-full" ...>...</aside>
</div>

{# Auto-dismissing toast #}
<div x-data="{ show: true }" x-init="setTimeout(() => show = false, 5000)" x-show="show" role="status" aria-live="polite">...</div>

{# Collapse — uses @alpinejs/collapse plugin #}
<div x-data="{ open: false }">
  <button @click="open = !open">Toggle</button>
  <div x-show="open" x-collapse>...</div>
</div>
```

`x-cloak` is styled in `app.css` `@layer base` (`display: none !important`) so partials don't flash on first paint.

## Forms

Two paths, both supported:

1. **Component partial** — `{% include "components/input.html" with field=form.titulo %}` — preferred for new templates; carries label, error association via `aria-describedby`, `aria-invalid` on error, focus ring.
2. **Raw `{{ form.field }}`** — works because `app.css` `@layer base` ships fallback styles for unclassed `<input>`/`<textarea>`/`<select>` (Tailwind v4 Preflight strips browser defaults; without the fallback, raw widgets render invisible). Use this when integrating with Django's `as_p` / `as_widget` flows or formsets.

Form contract:

```html
<form method="post" novalidate class="space-y-5">
  {% csrf_token %}
  {% if form.non_field_errors %}
    {% include "components/alert.html" with variant="danger" message=form.non_field_errors %}
  {% endif %}
  {# fields … #}
  <div class="flex flex-col md:flex-row gap-2 md:justify-end pt-2">
    <a class="...outline button..." href="...">Cancelar</a>
    <button type="submit" class="...primary button...">Guardar</button>
  </div>
</form>
```

Rules:
- `novalidate` on the form — let Django/server validate; HTML5 popups are inconsistent.
- Cancel on the left, primary on the right. Mobile: stack with primary on top (use `order-1`/`order-2`).
- Required indicator: `<span class="text-red-600 ml-0.5" aria-hidden="true">*</span>` after the label text.
- All `label`s, errors, and help text in **Spanish**.

## Tables

```html
<div class="overflow-x-auto rounded-lg border border-zinc-200 bg-white">
  <table class="w-full text-sm">
    <thead class="bg-zinc-50/60 text-left text-xs uppercase tracking-wider text-zinc-500">
      <tr>
        <th scope="col" class="px-4 py-2.5 font-semibold">Folio</th>
        <th scope="col" class="px-4 py-2.5 font-semibold hidden md:table-cell">Fecha</th>
        ...
      </tr>
    </thead>
    <tbody class="divide-y divide-zinc-200">
      {% for row in items %}
        <tr class="hover:bg-zinc-50">...</tr>
      {% empty %}
        <tr><td colspan="N">{% include "components/empty_state.html" with ... %}</td></tr>
      {% endfor %}
    </tbody>
  </table>
</div>
```

- Hairline borders, no Bootstrap-style striping (status colors do the heavy lifting).
- Hide low-priority columns at `<md` with `hidden md:table-cell`.
- Always provide an empty state (`{% empty %}`).

## Status, color, and accessibility

- WCAG 2.2 AA contrast: ≥4.5:1 body, ≥3:1 large text/UI components/borders.
- Status badges: text + color, never color-only (e.g. `<span>Aprobada</span>` inside the green pill).
- Visible focus ring on every interactive element: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-zinc-900` (or `outline-current` for inheritance).
- Touch targets ≥44×44 (24×24 hard floor per SC 2.5.8). Default button heights (`h-9`/`h-10`) meet this.
- `prefers-reduced-motion` respected globally via `@layer base` rule in `app.css`.
- `<main id="main">` plus a skip-link in `base.html` (Tailwind `sr-only focus:not-sr-only` pattern).
- One `<h1>` per page; headings in order; `aria-current="page"` on the active sidebar link.

## Responsive behavior

Mobile-first, Tailwind breakpoints: `sm 640` · `md 768` · `lg 1024` · `xl 1280` · `2xl 1536`.

- Navbar always visible; **sidebar hidden below `lg`** (mobile users open the drawer via the navbar hamburger button — `aria-label="Abrir menú"`).
- Tables wrapped in `overflow-x-auto`; low-priority columns hidden on mobile.
- Forms: single column on mobile, multi-column from `md` up via `grid grid-cols-1 md:grid-cols-2 gap-4`.
- Buttons in form footers: stacked on mobile (`flex flex-col gap-2`), inline + right-aligned on `md+` (`md:flex-row md:justify-end`).
- Verify at 320px width (no horizontal scroll outside `.overflow-x-auto`) and at 200% browser zoom.

## es-MX locale

- All user-facing copy in **Spanish**.
- Dates: `d/m/Y` for civil dates, `d/m/Y H:i` when timestamps matter.
- Currency: `MXN`.
- Names rendered as three fields (nombre, apellido_paterno, apellido_materno) per existing repo convention.

## Print / PDF

Tailwind is **not** used for WeasyPrint PDF templates. Those live in `app/templates/.../pdf/` (or per-app pdf folder) and use scoped inline `<style>` blocks tuned for paged media (`@page { size: Letter; margin: 1.5cm }`). Don't import `app.build.css` in a PDF template — it's tuned for the screen viewport.

## Quick checklist before merging UI work

- [ ] No purple/pink gradients, no glassmorphism, no emoji as icons
- [ ] All user-facing text in Spanish
- [ ] Renders at 320px width with no horizontal scroll
- [ ] Sidebar collapses below `lg`; drawer opens via navbar
- [ ] Tables wrapped in `overflow-x-auto`; low-priority columns hidden on mobile
- [ ] Every form input has a `<label>`; errors readable by screen reader (`aria-describedby`, `aria-invalid`)
- [ ] Visible focus ring on every interactive element
- [ ] Body contrast ≥4.5:1; status badges include text, not just color
- [ ] One `<h1>` per page; headings in order
- [ ] Empty states for every list/table
- [ ] Skip-link present and reachable on `base.html`
- [ ] Tested at 200% zoom
- [ ] No Bootstrap class names (`btn-`, `card`, `navbar`, `offcanvas`, `modal`, `alert-`, `badge text-bg-*`, `bi bi-*`) anywhere in `app/templates/` or `app/static/`
